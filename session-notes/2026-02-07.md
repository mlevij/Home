## Key Architecture Notes

### File Structure
- **`index.html`** — Landing page with ArcGIS map (popup rounding added this session)
- **`aes-data.html`** — Station data page (bulk of changes this session)
- **Service URL**: `https://services1.arcgis.com/KNdRU5cN6ENqCTjk/arcgis/rest/services/CSU_AES_Soils/FeatureServer`

### Field Mapping System
```javascript
FIELD_MAP     // short name → DCP 0-6" field name (e.g., 'Sand' → 'Sand_DCP_0_6_in')
ALLDEPTH_MAP  // short_AD name → AllDepth field name (e.g., 'Sand_AD' → 'Sand_AllDepth')
COMBINED_MAP  // merged map of both
```

### Custom Visualization Functions
- `renderPhScale(phVal, hostId, markerId)` — vertical pH column with animated marker
- `renderOmGauge(omAvg, omMax, hostId, needleId)` — SVG semicircle gauge with needle
- `renderBulletBars(stats, keys, hostId, suffix)` — horizontal bullet bars with threshold zones

### Reusable Design
All visualization functions accept optional `hostId`/`markerId`/`needleId` parameters so they can be instantiated for both the 0-6" and All Depths sections without code duplication.

## Lessons Learned
- **Always read the FULL original code before replacing sections** — replacing one chart accidentally wiped out unrelated features in the same rendering function
- **OneDrive + `.git` = corruption** — keep the git repo outside OneDrive
- **`.esri-widget` font override breaks Calcite icons** — target specific ArcGIS text classes instead
- **SVG gradient IDs must be unique** when multiple instances exist on the same page (e.g., two OM gauges)

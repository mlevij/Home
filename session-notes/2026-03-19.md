## Key Architecture Notes

### GIF Background Blending Order (GEE)
```
GIF_BG (#f5f2ec)
  .blend(satellite_vis)       ← satellite fills AOI; background shows outside
  .blend(boundary_vis)        ← county lines drawn on top in black
```
Export with `min: 0, max: 255, bands: ["vis-red", "vis-green", "vis-blue"]`

### GIF Layout in index.html
```
.gif-grid (2-col)
  ├── landsat_swco_timeseries.gif    (false color)
  └── landsat_swco_naturalcolor.gif  (natural color)

.gif-grid (2-col, justify-items: center)
  └── grandvalley_sentinel2.gif      (grid-column: 1/-1, max-width: 50%)
```

## Lessons Learned
- **`burn_year_labels` stacking**: only occurs when burn cell runs without a preceding fresh download — structuring each export cell as download + burn in one cell prevents it entirely
- **GEE video export background**: masked pixels outside the AOI clip region render as black by default; pre-visualizing and blending a solid background image before export is the cleanest fix — no PIL post-processing needed, county lines stay intact
- **GIF background via PIL near-black replace**: rejected because county boundary lines are the same black color as the background — can't distinguish them with a simple color threshold
